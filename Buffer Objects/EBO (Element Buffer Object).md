
The first step in optimisation, enable indexing vertex to avoid duplicates.
It define draw order to avoid drawing vertexes that are identical.

```c
float vertices[] = {  
    0.0f,  0.5f, 0.0f, // 0  top
    -0.5f, -0.5f, 0.0f, // 1  bot left
    0.5f, -0.5f, 0.0f, // 2  bot right
    -0.5f/2, 0.0f, 0.0f, // 3  mid left
    0.5f/2, 0.0f, 0.0f, // 4  mif right
    0.0f, -0.5f, 0.0f // 5  mid bot
};  
  
GLuint indices[] = {  // GLuint are OpenGL own unsigned int type (same can be done for float: "GLfloat")
    0, 3, 4,  // top triangle
    1, 5, 3,  // bot left triangle
    2, 5, 4  // bot right triangle
}; // I didn't draw each triangle in the same order (idk if order matter)
```

## Related Functions

### `glGenBuffers(n, buffers)` 

***Used to generate buffer(s)***

**n** : the numbers of buffers to generate.
**buffers** : an unsigned int or an array of unsigned int representing the id(s) of the generated buffers.

#### Example usage
```c++
// single EBO
unsigned int EBO;
glGenBuffers(1, &EBO);

// multible VBO
unsigned int EBOs[2];
glGenBuffers(2, EBOs)
```


### `glBindBuffer(target/type, buffer)`

***Used to assign a type to a generated buffer***

**target/type** : the type of buffer to bind (for EBO we use `GL_ELEMENT_ARRAY_BUFFER`).
**buffer** : the id of the targetted buffer.

#### Example usage
```c++
glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, id);
// the id here is the one generated by the glGenBuffers function

// so taking the above exemple it should be EBO or EBOs[i]
```

>[!hint]
> binding to 0 unbind the buffer.
> `glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, 0);`

### `glBufferData(target/type, size, indices, usage)`

***Used to fill the LAST BINDED buffer with vertices data***

**target/type** : the type of buffer to bind (for EBO use `GL_ELEMENT_ARRAY_BUFFER`)
**size** : the size of the indice array (basically just use a `sizeof()` on the indice array you send)
**indices** : an array of indice (for example a list of uint)
**usage** : the expected usage pattern of this data, most common being :
	- `GL_STREAM_DRAW` : the data is set only once and used by the GPU at most a few times.
	- `GL_STATIC_DRAW` : the data is set only once and used many times.
	- `GL_DYNAMIC_DRAW` : the data is changed a lot and used many times.

#### Example usage 
```c++
GLuint indices[] = {  
    0, 1, 3,  
    0, 2, 3  
};

glBufferData(GL_ELEMENT_ARRAY_BUFFER, sizeof(indices), indices, GL_STATIC_DRAW);
```
